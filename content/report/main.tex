\subsection{Substitution Matching}
The matching is done by initially matching the ouput tensor of the source of the axiom graph with a tensor of the output graph.
We then recursively match accoring to the following rules. For a given pair of nodes:
- If both are input...
- If src input and dst not input...
- If src not input and dst input...
- If neither is input, then they must be the same expression: we then recurively match on their subexpressions.
(We propagate a partial state as we go along).

\subsection{Substitution Application}

Our application system is split into 2 parts:
1) Apply
2) Canonicalize

1) Apply
We handle the following cases:
a) Surjective variable match. In particular, matmul(x,y) as src should match matmul(x,x)
b) Occurrence based rewriting. For example, given matmul(x,x), the system should be able to rewrite to matmul(x,x.T.T) (ie. only one application of x has been rewritten, so not the node).
c) Input and Output in the dst graph of the subsitution are aliased. For example, x.T.T -> x, should be able to be applied correctly.
x = input() #in
y = x.T
z = y.T #out
w = relu(z)

and get the following graph:

x = input()
w = relu(x)

d) A pattern must be able to match any subgraph.
For example, x.T.T should be albe to be applied even if x is not an input of the graph and x.T.T is not the output of the graph.

e) Must be able to match graphs with multiple outputs.

f) Should be able to handle dangling expressions cleanly:
- For example, consider:

x = input
y = tr(x)
z = tr(y)
w = matmul(y,z)

given x.T.T, we should be able to recover:

x = input
y = tr(x)
w = matmul(y, x)

We then perform a canonialicazion pass which does the following:
a) Remove common subexpressions
b) Remove unused tensors (except inputs, which must always be kept, and outputs, which must always be kept).
c) Remove graphs that became cycles.


One of the main motivations for enforcing a bidireactioanl rewrite system, was, in addition to completeness, the follows:
A nice property of a fully bidirectional rewrite system is that if we have an intermediate state s for which we have sound derivations src*->s and dst*->s, then we can invert the dst->s derivation and then chain it to obtain a sound src*->dst derivation (and viceversa).
GIevn that we are doing combinanorial searhc, this makes the search significnatly faster. Suppose that src and dst are a minium of n rewirets apart, and that each state can be rewritten to x new states (and assue, for simplifty, that these strates had not been visited before). Then, to reach dst from src, we would need to visit at least x^n states (assuming BFS). With the bidirectional approach, we can however intitiate the search from both src and dst, and thus visit at most x^(n/2) states. (We explore until we discover a commomn state.)
